Multi-Agent Productivity OS

One brain.
Many agents.

NEXUS coordinates specialist AI agents — each owning a domain — to manage your tasks, calendar, research, and memory in a unified flow.

01 / Core

The NEXUS node.
The conductor.

At the heart of the system sits a single orchestrator — NEXUS. It receives your intent, decomposes it into specialist tasks, and coordinates the entire swarm in parallel phases. Nothing happens without it deciding who to call and when.

Orchestrator Phase Planning Swarm Consensus

02 / Memory

Mnemo.
Four-layer recall.

NEXUS connects to Mnemo — the memory specialist. Every preference, fact, and semantic signal you produce is indexed across four layers: a reactive Blackboard, session JSON on disk, Supabase context items, and a pgvector semantic vault. Nothing is forgotten, everything is searchable.

Mnemo L1 Blackboard L3 Supabase Context L4 Vector Search

03 / Tools

Chrono.
Controls your calendar.

When your prompt involves time — meetings, deadlines, focus blocks — NEXUS routes to Chrono. Via MCP, Chrono calls Google Calendar, detects conflicts, and books time autonomously. No copy-pasting. No tab-switching. It also reads Atlas findings off the Blackboard to schedule related work.

Chrono Google Calendar MCP Conflict Detection Auto-scheduling

04 / Intelligence

Atlas.
Research at depth.

Atlas synthesises web intelligence, Wikipedia, and your knowledge base into structured briefings. It writes findings back to the Blackboard so Sage and Chrono can act on them in the same session — zero repetition, zero context loss.

Atlas Tavily Search Wikipedia MCP Blackboard Write

05 / Full Swarm

All agents.
One command.

On complex tasks NEXUS deploys the full specialist swarm in parallel phases. Atlas researches while Mnemo recalls context. Then Sage structures the output while Chrono schedules next steps. All coordinated, all reading the shared Blackboard, all streamed live to your screen.

Full Swarm Parallel Phases Shared Blackboard SSE Live Stream
