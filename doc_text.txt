This separation of identity from infrastructure is what makes the system maintainable and extensible.Mnemo — The Memory AgentMnemo is unique among the agents. It never speaks unless asked. It is always present in the right rail of the UI, always watching. After every interaction, Mnemo runs an importance scoring step (1–5) and commits only high-value information to long-term memory. It maintains all 4 memory layers and surfaces relevant past context to the orchestrator at the start of every new session.Layer 1 — session.state: working memory for the current workflow onlyLayer 2 — Firestore daily log: every event from today, retained for 30 daysLayer 3 — Long-term profile: curated facts, preferences, and patterns, permanentLayer 4 — Vector semantic search: note embeddings, recalled by meaning not keywords5. When a user clicks any completed MCP card in the left rail, a detail drawer opens showing exactly what that agent did — not just which tool was called. The drawer shows: (1) which agent triggered it and why, (2) each step taken with the exact API call made, (3) conflicts detected and resolved, (4) data written to session.state for the next agent, and (5) what Mnemo saved to long-term memory from this interaction. This answers the judging criterion for auditability and explainability.6.3 The Typing MomentThe most important UX decision in NEXUS is what happens the instant the user starts typing. Three simultaneous subtle changes occur: the input bar expands slightly, the agent coordination graph fades in from zero opacity in the center panel, and all 9 agent cards shift from &apos;Ready&apos; to an alert color state. Nothing is animated yet — just a color shift. The full system activates only on send.
